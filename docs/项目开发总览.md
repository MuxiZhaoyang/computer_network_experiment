## 🔄 开发流程（重要！）

### 第一阶段（2-3天）：基础设施
**目标**：完成消息分发和基础通信

#### 优先级1：MessageDispatcher
- **负责人**：成员一 + 成员二
- **任务**：
  1. 成员一实现MessageDispatcher（已完成框架）
  2. 成员二编写测试代码
  3. 联合调试确保UDP收发正常
- **验收标准**：
  - UDP socket创建成功
  - 能够接收和分发消息
  - 所有信号正常触发
- **其他人**：阅读文档，熟悉接口

#### 优先级2：NetworkDiscovery + MemberManager
- **负责人**：成员一 + 成员五
- **任务**：
  1. 成员一实现网络发现
  2. 成员五实现组员管理
  3. 两者配合测试
- **验收标准**：
  - 能够发现其他客户端
  - 成员列表正确更新
- **依赖**：MessageDispatcher完成

### 第二阶段（2-3天）：消息功能
**目标**：完成所有消息传输功能

#### 并行开发：
- **成员七**：MessageP2P（一对一消息）
- **成员三**：MessageBroadcast（广播消息）
- **成员六**：MemberRefresh（手动刷新）

#### 注意事项：
- 这三个模块相对独立，可以并行开发
- 都依赖MessageDispatcher
- 成员三需要等成员五的MemberManager完成成员列表同步

### 第三阶段（3-4天）：文件传输和界面整合
**目标**：完成文件传输和UI集成

#### 优先级1：FileTransfer
- **负责人**：成员四
- **任务**：实现TCP文件传输
- **验收标准**：
  - TCP服务器正常工作
  - 文件能够完整传输
  - 进度更新正确

#### 优先级2：MainWindow整合
- **负责人**：成员二
- **任务**：
  1. 实现init_modules()
  2. 实现connect_signals()
  3. 实现所有槽函数
  4. 美化界面
- **依赖**：所有其他模块基本完成

### 第四阶段（2-3天）：联调和优化
**目标**：完整测试和bug修复

#### 全体参与：
1. **集成测试**：按照测试指南测试所有功能
2. **Bug修复**：记录并修复发现的问题
3. **性能优化**：提升响应速度和稳定性
4. **文档完善**：补充注释和使用说明

## 🐛 常见问题快速解决

### Q1: 模块间如何通信？
**A**: 通过MessageDispatcher的信号。参考 `接口对接检查清单.md` 中的连接关系。

### Q2: 如何测试我的模块？
**A**: 参考 `详细测试方法指南.md`，每个模块都有具体的测试方法。

### Q3: 我的模块依赖其他模块怎么办？
**A**: 
1. 先模拟依赖模块的接口
2. 等依赖模块完成后再联调
3. 或者与负责依赖模块的同学一起开发

### Q4: 遇到bug怎么办？
**A**:
1. 检查日志输出
2. 使用print调试
3. 查看 `问题排查指南` 部分
4. 在群里求助

### Q5: 代码提交冲突怎么办？
**A**:
```bash
git pull
# 解决冲突
git add .
git commit -m "解决冲突"
git push
```

## 💡 最佳实践

### 代码规范
1. 函数和类添加文档字符串
2. 使用类型注解
3. 变量名清晰明确
4. 避免魔法数字（用常量）

### 测试规范
1. 每完成一个函数就测试
2. 写测试用例保存起来
3. 记录测试结果
4. 发现bug立即修复

### 协作规范
1. 提交前先拉取最新代码
2. 提交信息要清晰
3. 不修改别人的代码（除非协商）
4. 及时沟通进度和问题
